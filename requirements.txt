# Python deps for the Jetson side of the IoT Desk Companion.
# NOTE: preliminary — not version-pinned yet. The Jetson runs Python 3.6.9 (system);
# some of these may need a newer Python (venv / pyenv / Docker) — verify on-device
# via scripts/jetson_bootstrap.sh before pinning.

# --- UI: PyQt5 dashboard on the 4.3" HDMI DWIN screen ---
# Prefer the apt package on the Nano (python3-pyqt5); pip wheel may not exist for py3.6/aarch64.
PyQt5

# --- GPIO: relays, PIR, MQ2 digital, rotary encoder, buzzer ---
Jetson.GPIO

# --- DHT11/DHT22 temp+humidity (digital 1-wire on pin 7) ---
adafruit-circuitpython-dht        # needs adafruit-blinka; verify py3.6 support on-device

# --- MQ2 analog level via ADS1115 I2C ADC (channel A0, addr 0x48) ---
adafruit-circuitpython-ads1x15

# --- WS2812 LED strip on SPI0 MOSI (pin 19) ---
# NOT rpi_ws281x (that's Raspberry-Pi-only). Plain neopixel bit-bangs too jittery on Jetson.
adafruit-circuitpython-neopixel-spi

# --- pulled in by the adafruit-circuitpython-* libs ---
adafruit-blinka                   # board/busio shims; the py3.6 question lives here

# --- notifications ---
requests
python-telegram-bot               # only if we send Telegram alerts ourselves rather than via NemoClaw channels

# --- NemoClaw is NOT a pip package ---
# Installed separately:  curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash
# Our code talks to it via the `nemoclaw` / `openclaw` CLI (or its local gateway).
